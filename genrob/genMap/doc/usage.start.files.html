Start-Dateien

Zur Vereinfachung des Startens von genRob®-genControl mit genRob®-genMap als Modul liegen die Dateien run.bat (für Windows™) bzw. run (für Unix®-/Linux-/MacOS®X-sh) bereit.  Diese Start-Dateien sind möglicherweise für Test-Zwecke schon ausreichend, jedoch ist es in einigen Fällen notwendig, Parameter zu verändern.

Sofern beim Start der Name der Start-Datei nicht in einem Terminal(fenster) als Kommando eingegeben wurde, wird vom Betriebssystem für die Ein- und Ausgaben der jeweiligen Anwendung(en) ein Terminalfenster geöffnet.

Windows™

Unter Windows™ ist darauf zu achten, daß die Umgebungsvariable path passend gesetzt ist.  Es muß java.exe gefunden werden können.

Da die Start-Datei schon mit der Endung .bat vorliegt, kann sie sofort benutzt werden.  Arbeitsverzeichnis muß jeweils das Verzeichnis sein, in dem die Datei sich befinden.

Unix®/Linux/MacOS®X

Neben der Tatsache, daß das Programm java im Suchpfad für Programme zu finden sein und das Arbeitsverzeichnis das sein muß, welches die Datei beherbergt, gibt es hier weitere Möglichkeiten, wie die Arbeit angenehmer gestaltet werden kann.

In der ausgelieferten Version muß die Datei folgendermaßen benutzt werden: 

sh  run

Will man sich die Angabe der Shell sh ersprachen, so genügt ein einmaliges 

chmod +x run
um die Ausführung der Shell implizit werden zu lassen.

Sofern sich die Datei nun noch im Suchpfad befindet, läßt sie sich einfach über die Angabe 

run
starten.
