An dieser Stelle wird der Start der Java™-Anwendung genRob®-genControl/-genMap ohne den Umweg über eine Start-Datei beschrieben.  Trotzdem sollten diese Dateien parallel zur Veranschaulichung des folgenden benutzt werden.

genRob®-genMap läuft als Modul unter Kontrolle von genRob®-genControl.  Dazu muß beim Start von genRob®-genControl die Klassenbibliothek genRob.genMap.jar des Moduls im Klassenpfad enthalten sein und gleichzeitig als Argument (nicht Java™-Eigenschaft!) die Modul-Klasse genRob.genMap.ModuleImpl übergeben werden: 

java  -classpath ...genRob.genMap.jar...
	... genRob.genMap.ModuleImpl

Da genRob®-genControl mit mindestens der Java™-Eigenschaft genRob.genControl.port zur Angabe des TCP-Ports parametrisiert werden muß, ist die obige Beschreibung nicht ausreichend.  Hinzu kommt, daß von genControl die Jar-Datei und die zum Start zu benutzende Klasse angegeben werden muß.

Angenommen die Dateien des Pakets genRob®-genMap befinden sich im Pfad mypath und man will den Server an TCP-Port 6000 betreiben, so ist folgendes zu schreiben (hier für Windows™;  alles in einer Zeile;  der angegebene Klassenpfad ohne Leerzeichen): 

java
	-DgenRob.genControl.port=6000
	-classpath
		mypath\genRob.genControl.jar;
		mypath\genRob.genMap.jar
	genRob.genControl.Main
	genRob.genMap.ModuleImpl
Lesen Sie weitere Informationen zur Parametrisierung von genRob®-genMap.  Mehr Informationen zu Server-bezogenen Parametern, wie die oben erwähnte Server-Port-Angabe, sind in der Dokumentation von genRob®-genControl zu finden.
