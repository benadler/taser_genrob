Liste der Änderungen an der Software

Die Liste dokumentiert nicht nur die Änderungen an den Schnittstellen und deren Implementierungen, sondern auch Änderungen im Modul für der Roboter.

 0.7.4 → 0.8
  • Umstellung auf genRob®.genControl (Version 5.1).
 0.7.3 → 0.7.4
  • Ergänzungen und Bugfixes im Paket uhh.fbi.tams.mobilerobot.client
 0.7.2 → 0.7.3
  • überarbeitung der Dokumentation (neues Layout, weitere Informationen, etc.).
  • Scripte zum Start der Test-Anwendungen.
  • Neues Paket uhh.fbi.tams.mobilerobot.client. Mit den Klassen aus diesem Paket lassen sich schnell Client-Anwendungen für den mobilen Roboter schreiben, ohne das man Roblets oder Netzwerk-Kommunikation schreiben muss.
  • Automatische Kalibrierung über neue Einheit
  • Der Roboter wird nicht mehr automatisch kalibriert.
 0.7.1 → 0.7.2
  • Die Pose des Roboters wird zu Beginn geladen und beim Beenden des Moduls gespeichert.
  • Der Roboter wird nicht mehr automatisch kalibriert.
 0.7 → 0.7.1
  • IllegalArgumentException in AutoCalibration, wenn keine Scanpunkte von der Robotersteuerung mobiled empfangen wurden.
  • Neues Testprogramm DriveSquare im Paket uhh-fbi.tams.mobilerobot.test.
 0.6.1 → 0.7
  • Neue Methode turn() in der Einheit Motion.
  • Neue Ausnahme CommunicationException als Oberklasse der Ausnahmen ProtokollException, SocketCommunicationException und UnknownCommandException, die bei der Kommunikation mit der Robotersteuerung mobiled auftreten können.
 0.6 → 0.6.1
  • Abhängigkeiten zum Paket uhh.fbi.tams.utils entfernt. Paket wird nicht mehr benötigt.
  • Neue Klasse Convert zur Umrechnung in Hexadezimal- und Binärdarstellung. Diese Klasse war vorher im Paket uhh.fbi.tams.utils.
  • Umstellung auf genRob.genControl 4.0.2.
  • Zusätzliche Debug-Ausgaben in Commands.
 0.5.1 → 0.6
  • Neue automatische Kalibrierung: Der Roboter bestimmt zu Beginn seine Pose global anhand der inital vorgegebenen Marken Positionen und den gerade sichtbaren und gescannten Marken.
  • Logging-Informationen können durch das neue Schlüsselwort calibration erzeugt werden.
  • Wird dieses Logging eingeschaltet, bestimmt der Roboter seine Pose in regelmäßigen Abständen (5 Sekunden). Die Logging-Informationen enthalten die Abweichung zwischen der Pose-Schätzung des mobiled und der globalen Lokalisierung.
 0.5 → 0.5.1
  • Umstellung auf genRob.genControl 4.0.1 (siehe www.genRob.de).
  • Die Methode getAccess() der Einheit Access liefert jetzt auch true zurück, wenn ein Roblet®, das den Zugriff bereits hat, diesen nochmal prüft. Vorher wurde fälschlicherweise false zurückgegeben.
 0.4.1 → 0.5
  • Umstellung auf Version 2.0 der Roblet®-Bibliothek (siehe www.roblet.org).
  • Umstellung auf genRob.genControl 4.0 (siehe www.genRob.de).
  • Umstellung auf uhh.fbi.tams.utils 0.3.
  • Verknüpfung der Dokumentation mit der API von genRob.genControl 4.0.
  • Verknüpfung der Dokumentation mit der API der Version 2.0 der Roblet®-Bibliothek.
 0.4 → 0.4.1
  • Die Methoden getScan() und getMarks() der Klasse Laserscan liefern nie null zurück sondern immer eine Liste, auch wenn diese möglicherweise die Länge 0 hat.
  • Fehlerbereinigungen und Ergänzungen in der Dokumentation.
  • Link zum Kommunikations-Protokoll zur Robotersteuerung mobiled in der Dokumentation von ModuleImpl hinzugefügt.
 0.3.7 → 0.4
  • Umstellung auf genrob.genControl 3.0.
  • Erzetzen der Einheitenimplementierungen: Keine ERU- und MRU-Einheiten mehr, sondern nur noch eine Implementierung.
  • Neue Einheiten:
    • Access - Regelt den Zugriff auf die Steuerung des Roboters
    • Brakes - Zugriff auf die Bremsen des Roboters
  • Verzicht auf RobletException, die Methoden der Einheiten lösen jetzt paketeigene Ausnahmen aus, die die Fehlerursache klarer beschreiben.
  • Verzicht auf Klassen die nicht im Paket definiert werden wie z.B. Scan und Spot aus dem Paket genRob.genSimulation.unit. Entsprechend Einfürung neuer Klassen im Paket uhh.fbi.tams.mobilerobot.unit.
  • Die Klassen aus dem Paket uhh.fbi.mobilerobot.datatypes sind alle ins Paket uhh.fbi.tams.mobilerobot.unit verschoben. Dadurch genügt für die Entwicklung einer Roblet®-Anwendung nur letzteres Paket.
  • Umbenennung des Pakets uhh.fbi.tams.mobilerobot.unit.sensors in uhh.fbi.tams.mobilerobot.unit.sensor
  • Umbenennung des Pakets uhh.fbi.tams.mobilerobot.tests in uhh.fbi.tams.mobilerobot.test
  • Test- bzw. Beispielprogramme aus dem Paket uhh.fbi.tams.mobilerobot.test überarbeitet und angepasst.
  • Anpassung, Erweiterung und Überarbeitung der Dokumentation.
  • Dokumentation verlinkt mit der API-Dokumentation der Java™ 2 Standard Edition v1.4.2.
  • Dokumentation verlinkt mit der API-Dokumentation von genRob.roblet (Version 1.0).
  • Dokumentation verlinkt mit der API-Dokumentation von genRob.genControl (Version 3.0).
  • Der Release wird von nun an als ZIP-Datei komprimiert, da ZIP-Dateien standardmäßig auf den meisten Betriebssystemen entpackt werden können, ohne das weiter Software notwendig ist.
  • Versionsinformationen in Manifest-Datei der jar-Archive.
 0.3.6 → 0.3.7
  • Neue Methode getScanRadial() in der Einheit Laserscanner.
  • Anpassen der Dokumentation der Einheit Laserscanner.
  • Umstellung auf uhh.fbi.tams.utils Version 0.2.1.
