Ausführung

 genRob®-genView kommt normalerweise ohne Kommandozeilen-Parameter aus.  Dabei wird es folgendermaßen gestartet: 
java  -jar genRob.genView.jar
Die Dateien genView.bat und genView.sh enthalten im wesentlichen diesen Befehl und wurden zur Vereinfachung des Programmstarts unter Microsoft-Windows™ bzw. Unix®/Linux/MacOS™ beigegeben.

Unter der MacOS™-Oberfläche und bei geeigneten Installation auch unter Windows™ und bestimmten Linux-Oberflächen, läßt sich genRob®-genView auch über ein Doppelklicken auf die Datei genRob.genView.jar starten.

Lesen Sie auch Informationen zu möglichen Parametern.

Parameter

 genRob®-genView kommt normalerweise ohne Kommandozeilen-Parameter aus.  Unter Ausführung ist dazu mehr zu erfahren.

In manchen Fällen kann jedoch die Nutzung von Parametern sinnvoll oder gar notwendig sein.  In den nachfolgenden Abschnitten werden folgende Themen und die zugehörigen Parameter beschrieben:

  • OpenGL
  • Kamera
  • Skalen
  • genPCAN
  • BuR-Speicher
  • Meßfahrtprüfung
  • Persistenz
  • Konsole
  • Logging

Bei den Beispielen ist darauf zu achten, daß alles stets auf einer Kommandozeile einzugeben ist.

OpenGL

 Über die Java™-System-Eigenschaft sun.java2d.opengl und deren Setzen auf true kann die grafische Bibliothek des Java™-JDK ab Version 1.5 dazu aufgefordert werden, OpenGL zu benutzen und damit teilweise drastische Geschwindigkeitsgewinne bei der Darstellung hervorzurufen.  Dies ist auf Systemen von Bedeutung, die keine andere Beschleunigung benutzen (wie z.B. DirectX unter Windows™) und verbessert deutlich die Darstellung von Transparenzen.

Beispiel: 

java  -Dsun.java2d.opengl=true
         -jar genRob.genView.jar

Kamera

 Über die Java™-Eigenschaft genRob.genView.vlCamera.url kann eine URL auf einen Kamera-JPEG-Server gesetzt werden.  Die Bilder werden dann von genView zyklisch dargestellt.  Beispiel (in eine Zeile): 
java
    -DgenRob.genView.vlCamera.url=http://10.0.1.4:999
    -jar genRob.genView.jar

Temperatur- und Feuchteschieberegler

 Über die Java™-Eigenschaften genRob.genView.Scale.MinGrad und genRob.genView.Scale.MaxGrad können die Randwerte für die Temperaturskala des Sitzungsmodus gesetzt werden.  Diese Randwerte erscheinen in der Anzeige ganz links (MinGrad) bzw. rechts (MaxGrad).

Die Werte sind in Grad Celsius (°C) und als Fließkommazahl in englischer (!) Schreibweise anzugeben.  Das bedeutet, daß in Wahrheit eine Fließpunktzahl geschrieben werden muß - also 40.8 statt 40,8.  Beispiel: 

java  -DgenRob.genView.Scale.MinGrad=10.5
      -DgenRob.genView.Scale.MaxGrad=45.0
         -jar genRob.genView.jar
Analog erfolgt die Einstellung der Feuchte über die Java™-Eigenschaften genRob.genView.Scale.MinHumidity und genRob.genView.Scale.MaxHumidity, wobei die Werte die Einheit Prozent relativer Luftfeuchtigkeit haben. 
java  -DgenRob.genView.Scale.MinHumidity=10.0
      -DgenRob.genView.Scale.MaxHumidity=85.0
         -jar genRob.genView.jar

genPCAN

 Über die Java™-Eigenschaft genRob.genView.genPCAN kann der genPCAN-Roblet®-Server gesetzt werden.  Beispiel: 
java  -DgenRob.genView.genPCAN=localhost:8000
         -jar genRob.genView.jar
Über die Java™-Eigenschaft genRob.genView.genPCAN.properties muß dann die Konfiguration des CAN gesetzt werden.  Beispiel: 
java  -DgenRob.genView.genPCAN.properties=../CAN.prop
    		-jar genRob.genView.jar
Wie eine solche Datei auszusehen hat, ist bei genRob®-genPCAN beschrieben.

Über die Java™-Eigenschaft genRob.genView.genPCAN.bSetDateTime kann für eine genPCAN-Linux™-Maschine bestimmt werden, daß Datum und Uhrzeit auf die gleichen Werte, wie die der genView-Maschine gesetzt werden.  Der Wert muß dann true sein.  Beispiel: 

java  -DgenRob.genView.genPCAN.bSetDateTime=true
         -jar genRob.genView.jar
Über die Java™-Eigenschaft genRob.genView.genPCAN.bCorrectPackages wird die zeitliche Korrektur von CAN-Paketen auf dem genPCAN-Server gesteuert.  Für eine Korrektur muß der Wert true sein.  Beispiel: 
java  -DgenRob.genView.genPCAN.bCorrectPackages=true
         -jar genRob.genView.jar

BuR-Speicher

 Über die Java™-Eigenschaft genRob.genView.BuR.store kann der BuR-Speicher-Roblet®-Server gesetzt werden.  Beispiel: 
java  -DgenRob.genView.BuR.store=localhost:10000
         -jar genRob.genView.jar
Wird der Server gesetzt, so wird standardmäßig als Ort der Meßdaten home benutzt, dem Standardort von BuR-Speicher.  Befindet man sich an einem anderen Ort, so setzt man diesen per Java™-Eigenschaft genRob.genView.BuR.store.location.  Beispiel: 
java  -DgenRob.genView.BuR.store=localhost:10000
         -DgenRob.genView.BuR.store.location=center1
         -jar genRob.genView.jar

Meßfahrtprüfung

 Über die Java™-Eigenschaft genRob.genView.session.deviation kann die maximale Abweichung in Prozent von Meßfahrten hinsichtlich der gemessenen und kartierten Länge gesetzt werden.  Beispiel: 
java  -DgenRob.genView.session.deviation=10.0
         -jar genRob.genView.jar
Im obigen Beispiel wird eine maximale Abweichung von 10% eingestellt.  Ist die Java™-Eigenschaft nicht oder fehlerhaft gesetzt, so werden 5% angenommen.

Persistenz

 Mit Hilfe der Java™-Eigenschaft genRob.genView.serialize kann die Serialisierungsdatei von genView-Daten festgelegt werden, damit die Daten beim nächsten Anwendungsstart wieder zur Verfügung stehen.  Beispiel: 
java  -DgenRob.genView.serialize=../genView.data
         -jar genRob.genView.jar
Läßt man die Angabe weg, so wird als Dateiname genRob.genView.serialized im genView-Startverzeichnis angenommen. 

Konsole

 Über das Setzen der Java™-Eigenschaft genRob.genView.console kann die genView-Konsole aktiviert werden.  Diese Funktionalität ist für Entwickler gedacht und ermöglicht momentan die Anzeige der in genView laufenden Threads.  Beispiel: 
java  -DgenRob.genView.console=
         -jar genRob.genView.jar
Details zur Nutzung der Konsole finden sich hier.

Logging

 genRob®-genView kann für Entwickler viele Informationen über seine Aktivitäten auf der Konsole ausgeben.  Diese Ausgaben sind standardmäßig abgeschaltet.

Um Informationen auf der Konsole ausgeben zu lassen, muß die Java™-Eigenschaft genRob.genView.log bestimmte Schlüsselworte enthalten.  Will man die intern benutzte genRob®-genControl-Klienten-Bibliothek zu Ausgaben bewegen, muß die eben genannte Eigenschaft das Schlüsselwort client enthalten und außerdem muß die Eigenschaft genRob.genControl.client.log mit weiteren Schlüsselworten versehen werden (s.u.).

Folgende Schlüsselworte gibt es:

  • camera - Kamera-Funktion
  • client - genRob®-genControl-Klienten-Bibliothek
  • component - Ereignisse der Swing®-Komponenten
  • figletFocus - Fokus
  • figletRegistrar - Figlet-Verwaltung
  • genPCAN - Kommunikation zur genRob®-genPCAN-Instanz
  • jini - Jini™-Subsystem
  • main - Start, Konsole
  • mlItem - Kartenelemente/Karten-Server
  • robotBase - Roboter-Basis inkl. Typ-Anzeige
  • robotControl - Anwendung-zu-Roboter-Steuerung
  • robotForeign - Anwendung-zu-Roboter-Kommunikation
  • robotOrigin - Roboter-zu-Anwendung-Kommunikation
  • robotState - Roboter-Zustandsmaschine
  • session - Sitzungsfunktion
  • swingContent - Fensterinhalt
  • swingFrame - Fenster inkl. seiner Serialisierung und Drucken
  • swingPaint - Verarbeitung des Zeichnen-Ereignisses
  • tracker - PersonTracker-Funktion
  • worldBase - Basis des Weltteils (Karte, Roboter etc.)
  • worldTransponder - Transponder-Verwaltung

Hier ein Beispiel: 

java  -DgenRob.genView.log=main,jini
         -jar genRob.genView.jar

Läßt man die Eigenschaft leer (genRob.genView.log=), so ist das gleichbedeutend mit dem Weglassen der Eigenschaft.

genRob®-genControl-Klient-Bibliothek

Folgende Schlüsselworte gibt es für das Logging des Klienten:
  • base - Basisinformationen
  • call - Aufruf und Verarbeitung entfernter Funktionalität
  • confirm - Bestätigungen der Transportebene mit einem Server (nur zusammen mit transport)
  • clientConnection - Verbindungsaubau und -halt einer RMI-Verbindung
  • connect - Verbindungsauf- oder abbau
  • genMediator - genMediator-Verzeichnisdienst
  • httpdInform - generelle Informationen zum HTTP-Server
  • httpdExplain - Download-Informationen des HTTP-Server
  • httpdHint - Detail-Informationen des HTTP-Server
  • link - Verbindungsebene mit einem Server
  • load - Laden von Klassen und Ressourcen vom Server
  • queues - Paketverwaltung der Verbindung mit einem Server
  • sieve - Sieb-Aktivitäten
  • stream - Ströme mit einem Server
  • transport - Transportebene mit einem Server
  • tube - Röhrenebene mit einem Server
Dabei ist zu beachten, daß für die Eigenschaft genRob.genView.log auch der Wert client gesetzt ist, da sonst die Klienten-Bibliothek nicht für die Ausgabe von Log-Informationen initialisiert wird.

Hier ein Beispiel: 

java  -DgenRob.genView.log=client,main
      -DgenRob.genControl.client.log=base
     -jar genRob.genView.jar

Läßt man die Eigenschaft leer (genRob.genControl.client.log=), so ist das gleichbedeutend mit dem Weglassen der Eigenschaft.
