Funktionsweise

 genRob®-genControl ist eine Java™-Anwendung (genRob.genControl.jar).  Als solche läuft das Programm in einer JVM (Java™ virtual maschine).

genControl ist weiterhin ein Roblet®-Server.  Als solcher bietet er eine Laufzeitumgebung für Roblets®.

Um Roblets® in den Server hinein zu bekommen, bietet genControl noch eine Netzwerkschnittstelle an.  Um diese Schnittstelle einfach bedienen bzw. überhaupt benutzen zu können, wird die sog. Klienten-Bibliothek (genRob.genControl.client.jar) mitgeliefert.

Anwendungen, die einen Roblet®-Server nutzen (können), werden Roblet®-Anwendung genannt.  Damit eine Roblet®-Anwendung einen Roblet®-Server (vom Typ genRob®-genControl) nutzen kann, braucht sie im Grunde nur die Klienten-Bibliothek verwenden.

Damit eine Roblet®-Anwendung eine spezielle Funktionalität eines Roblet®-Servers nutzen kann, braucht sie die Definition der von ihm bereitgestellten Einheiten.  Diese Definitionen sind in den jeweiligen Einheiten-Bibliotheken zusammengefaßt.  Auch für genRob®-genControl gibt es eine solche Bibliothek (genRob.genControl.unit.jar).  Letztere enthält alles, was an Administration des Servers selbst möglich ist.

Der Roblet®-Server genRob®-genControl ist so konstruiert, daß er um beliebige weitere Funktionaliät erweitert werden kann.  Dazu muß nur ein sogenanntes Modul geschrieben werden, welches die gewünschte Funktionalität anbietet.  Ein solches Modul muß die Modul-Schnittstelle von genRob®-genControl implementieren und kann dann beim Start eines Servers mit übergeben werden, so daß genControl das Modul laden und benutzen kann.  Jeder Server lädt immer mindestens das Modul, welches die Server-spezifische Administration ermöglicht. 

Jedes Modul muß die von ihm vermittels des Servers bereitgestellte Funktionalität in Form von Einheiten-Definitionen und daher auch in der Regel in Form einer eigenen Einheiten-Bibliothek veröffentlichen - mindestens jedoch den Anwendungen zur Verfügung stellen, die die Funktionalität nutzen sollen.

Damit der Server seine Netzwerkschnittstelle erzeugen kann, benötigt er ein TCP-Port.  Da keine Standart-Port-Nummer existiert, ist die Angabe eines Ports obligatorisch.
